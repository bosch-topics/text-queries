Access Request Instructions

Step 1: Initiating Access in OneIDM

For initiating the access in OneIDM, kindly click the link: https://oneidm.bosch.com/oneidm/html/rbwebshop/#/start/itshop/VI_Start

Step 2: Request Process

Click Request → New Request → Click the Select IT application → Search for PPM/011 & Portal IT application.

Step 3: Role Selection

Select the Filter option in "Services" and search for the required roles: PPM011 X:B0PU_SMPL_REQUESTER & Portal - CN=rb_my_i_buy_virtual_company_mall_user,OU=portal,OU=groups,OU=esi,DC=wam-bosch,DC=com. Select the roles and click "Add to Shopping Cart".

Step 4: Shopping Cart

Select the shopping cart symbol at the top right side of OneIDM → Select "Shopping Cart" → Click the option "Edit".

Step 5: Business Justification

Enter the detailed business justification in "Reason". Select the Validity date from start to end. Click "Save" → Click "Submit".

Step 6: Request Number

A combined request number will be generated after submission.

Step 7: Checking Status

To check the status in OneIDM, click Request → Find Request → Enter CRN in "Search by Request ID" and check the Status column. Initially, the status will be updated as "New", but once the status changes to "Assigned", kindly try to access the application after the status has changed to "Assigned".

If Access is Still Unavailable:

If you are unable to access the application, kindly register your request/query through the link: https://boschindia.service-now.com/sp?id=sc_cat_item&sys_id=ed35579747e1ce10750cc6de116d4319

If prompted, “Login with SSO” using NTID, or you can also create a ticket by navigating from the Employee Self Service portal → User Access Management → OneIDM Support Request.

Step 1: Create Ticket

Click "Request for" and select "Myself or Others".

Step 2: Beneficiary NT ID

Enter the "Beneficiary NT ID".

Step 3: Select Application

Select Application as "oneIDM".

Step 4: Select Module

Select Module as "PPM/011".

Step 5: Business Justification for Access

Enter the "Business reason" in the "Business justification for access".

Step 6: Describe the Access Issue

Enter the details about access or issue in "Describe access right / Tcode / Issue".

Step 7: Attach Screenshot

Add any error screenshot as an attachment.

Step 8: Submit the Request

Click "Submit", and a ServiceNow ticket will be created.

Step 9: Email Notification

An email notification will be triggered when the ticket is created, and the ticket status can be checked in the ServiceNow portal. Additionally, email notifications will be sent for every update or change to the ticket.

Step 10: Check Ticket Status

To check the status of the created ticket in the ServiceNow portal, kindly click the "Request" option. The created ticket number will be listed there, and the user can check the ticket status.
